Teknologi

Apa Itu RAG?

Retrieval Augmented Generation

RAG adalah teknik AI yang menggabungkan kemampuan pencarian informasi dengan generasi teks. Alih-alih hanya mengandalkan pengetahuan yang sudah ada di dalam model, RAG memungkinkan AI mengakses sumber data eksternal secara langsung sebelum menjawab.

Intinya: AI tidak menjawab dari ingatannya sendiri, tapi mencari fakta dulu di dokumen kamu, baru menjawab.

Berapa harga produk X?
Berdasarkan Price List 2024 (hal. 3), harga Produk X adalah Rp 2.500.000.

Apa itu Chunking?

Dokumen dipecah menjadi potongan-potongan kecil agar bisa di-embed secara individual dan dicari pada level granular

Apa itu Embedding?

Proses mengubah teks menjadi vektor angka berdimensi tinggi. Teks yang mirip punya vektor yang berdekatan di ruang angka

Kapan Dilakukan?

Sekali saat setup awal. Update hanya saat ada dokumen baru atau perubahan, tidak perlu retrain seluruh model

Vector DB: Cara Lama

Implementasi RAG konvensional yang kompleks

Pecah dokumen jadi chunk kecil (500-1000 token)

Ubah setiap chunk jadi vector embedding

Simpan di database terpisah (Pinecone, Weaviate, Chroma)

Cari chunk paling mirip dengan similarity search

Masalahnya: chunking menghilangkan konteks dan struktur dokumen

Problem

Masalah Vector DB

Kita butuh pendekatan yang lebih sederhana dan akurat

Kehilangan Konteks

Setup Kompleks

Maintenance Tinggi

Update dokumen berarti re-embed semua chunk terkait

Biaya Infrastruktur

Vector DB punya biaya sendiri di luar biaya LLM

Pendekatan Baru

PageIndex: Lebih dari Sekadar RAG

Menggunakan Model Context Protocol (MCP) untuk menghubungkan AI langsung dengan sumber data tanpa kompleksitas Vector DB

MCP sebagai Jembatan

AI bisa mengakses data real-time melalui protocol standar, bukan chunking dan embedding manual

Dokumen Tetap Utuh

Tidak perlu memecah dokumen. Struktur dan konteks terjaga sepenuhnya

Setup dalam Menit

Tidak perlu setup Vector DB, embedding pipeline, atau chunking strategy
