Latar Belakang

Masalah di Industri

  • Developer AI Agent harus setup sendiri: LLM hosting, sandbox, memory, tracing, storage — wiring yang fragmentasi
  • Web & Agent biasanya beda platform — management, domain, dan observability terpisah
  • Agent framework berkembang cepat, tapi infrastruktur untuk production masih manual

Kenapa Tencent Cloud Membuat Ini?

  • EdgeOne punya 3.200+ PoP global, 2.500+ di Asia
  • Expand dari infrastruktur pasif (CDN) ke platform developer aktif
  • Makers = all-in-one: hosting web + AI Agent + edge functions + storage

Timing yang Tepat

  • Produk masih fase free / pre-commercial
  • Tencent aggressively building adoption
  • Launch rewards 3M token
  • Creator rewards 50M token

Agent vs Cloud Functions

agents/

Conversational, multi-turn, stateful

cloud-functions/

Stateless, non-LLM, API endpoint

Optimalisasi Token

  • Session memory untuk kurangi pengulangan konteks
  • Pisahkan business logic dari LLM calls
  • Sandbox tools untuk operation berat (file, browser)

Keamanan

  • Credentials di context.env
  • Input validation di cloud-functions layer
  • Sandbox isolation otomatis

Observability

  • Manual instrumentation via context.tracer
  • Local debug: edgeone makers dev
  • Trace viewer di /agent-metrics

Tapi kapan sebaiknya menggunakan model built-in, dan kapan vendor API?

Built-in vs Vendor API

Model Built-in

  • Prototype / MVP — langsung jalan tanpa daftar API Key
  • Eksperimen — gratis 500K token/bulan
  • Aplikasi ringan — chat, Q&A, summarization
  • Benchmark — gonta-ganti model cepat

Vendor API (BYOK)

  • Model spesifik — GPT-4o, Claude Sonnet, Gemini
  • Production dengan SLA tertentu
  • Data compliance — tidak diproses pihak ketiga
  • Karakteristik performa model tertentu

Hybrid Strategy

P2

BYOK

Scale, specific model

P3

Hybrid

Built-in simple, BYOK complex

Selain komputasi dan AI, EdgeOne Makers juga menyediakan storage

Storage

KV Storage

Distributed variable — Key → Value

  • Max per value: 25 MB
  • Runtime: Edge Functions only
  • Consistency: Eventual (~60 detik)
  • Use case: Config, counters, session
await my_kv.put("visitCount", "1"); const count = await my_kv.get("visitCount");

Blob Storage

Distributed object — Path-based hierarchy

  • Max per object: 25 MB
  • Runtime: Edge & Cloud Functions
  • Consistency: Eventual (~60 detik)
  • Use case: Images, dokumen, AI content
import { getStore } from "@tencent/pages-blob"; const store = getStore("uploads"); await store.set("photos/avatar.jpg", data); const file = await store.get("photos/avatar.jpg");

Di balik angka-angka itu, ada filosofi desain yang menarik

Memahami Logika di Balik EdgeOne Makers

Developer fokus ke business logic, platform handle infrastructure

1

Decoupled Architecture

Agent RuntimeSandboxStorage

Masing-masing punya lifecycle & scaling sendiri. Micro-VM sandbox: lazy start, reuse, recycle saat timeout.

2

Framework-Agnostic by Design

Bukan "pake SDK kami" — adaptasi ke framework apapun. context.tools auto-format ke bentuk yang dimengerti framework target.

3

Session vs Request Mode

agents/ → Session

conversation_id routing, memory reusable, cocok multi-turn

cloud-functions/ → Request

Stateless, elastic scaling, cocok API & webhook

4

Zero-Intrusion Observability

Auto-detect framework & inject instrumentor. Local & cloud trace sharing — primitives identik.

5

Web + Agent = Satu Stack

Frontend (React/Next.js) + AI Agent = satu project, satu domain, satu deploy pipeline.
