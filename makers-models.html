Tools dalam Satu Platform

Agent Runtime

Hosting LLM calls + orchestration loop. Session-based routing. Max 30 menit/request.

Sandbox Tools

2 layer API: LLM tools (browser_*, files_*, shell) + Dev API (atomic). Micro-VM per sesi.

Edge Functions

Serverless JS di 3.200+ PoP. File-based routing. Ultra-low latency di node terdekat.

Cloud Functions

Node.js, Python, Go. Express/FastAPI/Gin. 128 MB package. Multi-region deploy.

Storage

KV — key-value, 25 MB/value, eventual consistency. Blob — object storage, directory hierarchy.

AI Gateway

1 API Key + 1 endpoint. Compatible OpenAI/Anthropic/Vercel AI SDK. Streaming SSE native.

Observability

Auto-collect traces, zero-intrusion. Local debug panel + cloud dashboard.

Semua tools di atas berada dalam satu project, satu domain, satu deploy pipeline — tidak perlu integrasi terpisah.

Best Practice

Struktur Project

project/
├── agents/              ← Session mode
│   ├── customer-service/
│   │   ├── index.ts     → POST /customer-service
│   │   └── stop.ts      → POST /customer-service/stop
├── cloud-functions/      ← Request mode
│   ├── api/users.ts
├── edge-functions/       ← Edge (ultra low-latency)
├── static/               ← Frontend assets
└── edgeone.json          ← Project config

Agent vs Cloud Functions

agents/

Conversational, multi-turn, stateful

cloud-functions/

Stateless, non-LLM, API endpoint

Optimalisasi Token

  • Gunakan session memory untuk mengurangi pengulangan konteks
  • Pisahkan business logic dari LLM calls → taruh di cloud-functions
  • Sandbox tools untuk operation berat (file process, browser)

Keamanan

  • Credentials simpan di context.env — jangan hardcode
  • Input validation selalu dilakukan di cloud-functions layer
  • Sandbox isolation otomatis oleh platform

Observability

  • Gunakan context.tracer untuk manual instrumentation
  • Local debug dulu via edgeone makers dev
  • Trace viewer di /agent-metrics saat development
Pemahaman mendalam tentang model dan konsep inti menjadi fondasi pengembangan yang efektif

Logika di Balik EdgeOne Makers

"Developer fokus ke business logic, platform handle infrastructure"

Decoupled Architecture

Agent Runtime, Sandbox, Storage — masing-masing lifecycle & scaling sendiri. Micro-VM: lazy start, reuse, recycle.

Framework-Agnostic

Bukan "pake SDK kami" — adaptasi ke framework apapun. Auto-format tools & store ke native tiap framework.

Session vs Request

agents/ → sticky session, stateful, multi-turn
cloud-functions/ → stateless, elastic, 1:1

Zero-Intrusion Observe

Auto-detect framework & inject instrumentor. Local & cloud trace sharing — primitives identik.

Web + Agent = 1 Stack

Frontend (React/Next.js) + AI Agent = satu project, satu domain, satu deploy pipeline.

Agent Runtime (LLM)
Sandbox (Tools)
Storage (Memory)
| Decoupled but unified via context
