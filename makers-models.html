Agent vs Cloud Functions

agents/ untuk conversational, multi-turn, stateful. cloud-functions/ untuk stateless, non-LLM, API endpoint.

Optimalisasi Token

Gunakan session memory untuk mengurangi pengulangan konteks. Pisahkan business logic dari LLM calls ke cloud-functions.

Keamanan

Credentials simpan di context.env. Input validation selalu di cloud-functions layer. Sandbox isolation otomatis.

Observability

Gunakan context.tracer untuk manual instrumentation. Local debug dulu via edgeone makers dev sebelum deploy.

Logika di Balik
EdgeOne Makers

"Developer fokus ke business logic, platform handle infrastructure"

Decoupled Architecture

Agent Runtime (LLM), Sandbox (Tools), dan Storage (Memory) masing-masing punya lifecycle dan scaling sendiri. Micro-VM sandbox: lazy start, reuse dalam sesi, recycle saat timeout.

Framework-Agnostic by Design

Bukan "pake SDK kami", tapi adaptasi ke framework apapun. context.tools auto-format ke bentuk yang dimengerti framework target. context.store auto-adapt ke native session type.

Session vs Request Mode

agents/ → Session Mode

conversation_id routing, sticky ke instance. Memory & state reusable. Cocok: multi-turn, long-running tasks.

cloud-functions/ → Request Mode

Stateless, 1 request = 1 instance. Elastic scaling, langsung direclaim. Cocok: API endpoint, webhook.

Zero-Intrusion Observability

Auto-detect framework dan inject instrumentor. Local & cloud trace sharing menggunakan primitives yang sama persis.

Web + Agent = Satu Stack

Frontend (React/Next.js) + AI Agent = satu project, satu domain, satu deploy pipeline. Tidak perlu lagi mengelola dua platform terpisah.
