Preliminary Assessment

Tantangan Klinis Saat Ini

Masalah di Lapangan

Diabetes sering kali tidak bergejala jelas pada tahap awal (silent killer). Banyak pasien datang sudah dalam kondisi komplikasi.

Keterbatasan waktu konsultasi dan jumlah tenaga medis membuat skrining massal menjadi tantangan besar.

Data yang Tidak Terpakai

Kita memiliki data rekam medis (gejala, riwayat keluarga) yang sangat berharga, namun sering hanya "tersimpan" tanpa dianalisis polanya.

Pertanyaan Kunci: Bisakah kita memanfaatkan data historis ini untuk memprediksi risiko pasien baru?
Proposed Solution

Machine Learning sebagai "Asisten Diagnosa"

KONSEP

Kita tidak menggantikan dokter. Kita membangun alat bantu (tools) yang mampu menganalisis 16 variabel gejala dalam hitungan milidetik untuk memberikan "Second Opinion" berbasis data.

Model ini akan belajar dari 520 riwayat kasus pasien sebelumnya, mencari pola yang mungkin terlewat oleh mata manusia.

Aplikasi biasa hanya menampilkan data. Aplikasi ML ini belajar dari data. Semakin banyak data masuk, semakin pintar ia dalam mengenali pola risiko.

Minggu 01: Tahap Anamnesis Data

Fokus pada pemahaman mendalam terhadap kondisi data.

Hari 1: Adiministrasi & Setup

PYTHON / GIT

Mempersiapkan "ruang praktik" digital. Install Python, VS Code, dan membuat Repository GitHub sebagai folder rekam medis digital.

Hari 2: Pengenalan Alat (Pandas)

PANDAS

Pandas adalah stetoskop kita. Kita belajar membaca file CSV (rekam medis), melihat struktur data, dan statistik dasar.

Hari 3-4: Analisis Riwayat (EDA)

MATPLOTLIB / SEABORN

Memvisualisasikan data. Apakah usia mempengaruhi risiko? Bagaimana distribusi gula darah? Kita cari polanya sebelum mengobati.

Hari 5-7: Triage Data (Cleaning)

PREPROCESSING

Menangani data yang hilang atau anomali. Layaknya triage di UGD, kita tentukan data mana yang "layak olah" dan mana yang harus dibersihkan.

Minggu 02: Tahap Intervensi & Evaluasi

Dari teori menjadi alat diagnosa yang fungsional.

Hari 8-9: Seleksi Gejala (Feature Eng.)

FEATURE SELECTION

Tidak semua gejala relevan. Kita gunakan teknik statistik untuk menemukan 5-6 gejala utama (Main Symptom) yang paling berkorelasi dengan diabetes.

Hari 10-11: Pelatihan Model

SCIKIT-LEARN

Menggunakan algoritma Random Forest dan XGBoost. Kita "melatih" mesin dengan 80% data pasien lama untuk mengenali pola baru.

Hari 12: Uji Efikasi (Evaluation)

METRICS

Menguji model dengan 20% data yang belum pernah dilihat (Test Set). Ukur akurasi, presisi, dan recall. Apakah model layak digunakan?

Hari 13-14: Praktik & Deployment

STREAMLIT

Membungkus model menjadi Aplikasi Web Streamlit. Hasil akhirnya adalah link website yang bisa dibagikan ke siapa saja.

Triage Data: Pembersihan

Menyiapkan Data Sebelum Analisis

"Data yang bersih adalah separuh dari diagnosa yang benar."

Proses Cleaning

Data medis sering tidak lengkap. Ada pasien yang lupa mengisi riwayat, atau ada kesalahan input.

1
Handling Missing Values

Mengisi data kosong dengan rata-rata (mean) atau modus, agar analisis tidak bias.

2
Encoding

Mengubah jawaban "Ya/Tidak" menjadi angka "1/0" agar mesin bisa membaca.

# Contoh Encoding
mapping = {
  "Yes": 1, "No": 0,
  "Male": 1, "Female": 0
}

df = df.replace(mapping)
03

Diagnosis Banding: Seleksi Fitur

Mencari Gejala Paling Signifikan

Dari 16 gejala, tidak semuanya memiliki dampak sama terhadap diabetes. Kita harus menemukan "Key Symptom" untuk mempermudah diagnosa.

Penelitian menunjukkan Polyuria (sering buang air kecil) dan Polydipsia (sering haus) adalah dua gejala dengan korelasi tertinggi. Jika dua ini muncul bersamaan, risiko diabetes meningkat signifikan.

Tingkat Pengaruh Gejala

Polyuria 92%
Polydipsia 88%
Sudden Weight Loss 75%
04

Protokol Pengobatan: Modeling

Memilih Algoritma Terbaik

Kita akan membandingkan dua pendekatan "diagnosa" oleh mesin:

Logistic Regression

Cocok untuk baseline. Mudah dijelaskan ke pasien karena mirip sistem skor risiko.

Random Forest / XGBoost

"Tim Dokter Spesialis". Menggunakan banyak pohon keputusan (second opinion) untuk hasil yang lebih akurat.

TRAINING PHASE

80% Data Latih

Model mempelajari pola dari 416 pasien.

20% Data Uji

Model diuji dengan 104 pasien baru yang belum pernah dilihat.

05

Efikasi Pengobatan: Evaluation

Mengukur Keberhasilan Model

"Akurasi tinggi tidak cukup. Dalam medis, kita tak boleh melewatkan pasien yang benar-benar sakit."

Metrik Utama: RECALL

Recall menjawab: "Dari semua pasien yang benar-benar sakit, berapa persen yang berhasil kita deteksi?"

Lebih baik model kita salah prediksi "Sakit" (pasien cemas tapi akhirnya sehat), daripada salah prediksi "Sehat" (pasien terlewatkan dan tidak dapat pengobatan). Ini disebut meminimalkan False Negative.

Confusion Matrix

TP
Deteksi Benar
FP
False Alarm
FN
MISSED (Bahaya!)
TN
Sehat Benar
06

Penyesuaian Dosis: Tuning

Optimasi Hyperparameter

Seperti menyesuaikan dosis obat, kita perlu mengatur parameter model (Hyperparameter) agar efektif maksimal. Kita tidak bisa pakai dosis standar untuk semua kasus.

  • Grid Search: Mencoba semua kombinasi dosis.
  • Cross Validation: Menguji efek obat ke berbagai kelompok pasien (fold).
# Contoh Tuning
params = {
  'n_estimators': [50, 100, 200],
  'max_depth': [5, 10, 20]
}

grid = GridSearchCV(model, params, cv=5)
