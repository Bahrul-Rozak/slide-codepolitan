Sesi 2 — 10 Menit

Apa Itu Sandbox?

Micro-VM terisolasi yang diberikan kepada setiap agent

Sesi Terisolasi

Setiap sesi agent mendapatkan sandbox sendiri. Agent A tidak bisa melihat file Agent B.

Lazy Start

Sandbox baru aktif ketika digunakan. Tidak ada resource yang terbuang saat idle.

Auto-Recycle

Sandbox otomatis dimatikan setelah 5 menit idle. Resource dikembalikan ke pool.

Fully Isolated

Tidak dapat mengakses host server. Semua operasi terbatas pada sandbox masing-masing.

Sesi 2 — Lanjutan

Layer Sandbox API

Tool API (untuk LLM)

browser_open(url) → Buka web
browser_text() → Baca konten
files_write(path, data) → Tulis file
files_read(path) → Baca file
shell_run(command) → Jalankan command
code_run(code, lang) → Eksekusi kode

Atomic API (untuk Developer)

// Shell ctx.sandbox.commands.run("ls -la") // File ctx.sandbox.files.write("data.txt", content) // Browser ctx.sandbox.browser.navigate(url) // Code ctx.sandbox.runCode( "console.log('hi')", "js" )

Tool API dipanggil oleh LLM melalui function calling. Atomic API dipanggil langsung oleh developer di kode.

Sesi 2 — Penutup

Mengapa Sandbox Penting?

Eksekusi Nyata

Agent menjalankan kode secara sungguhan — bukan simulasi. Hasil yang diperoleh adalah output asli.

Browsing & Scraping

Agent dapat membuka website, membaca halaman, mengekstrak informasi secara otomatis.

Proses File

Convert CSV ke JSON, resize gambar, merge dokumen — semua dilakukan di dalam sandbox.

Keamanan Terjamin

Semua operasi terisolasi. Tidak ada efek samping terhadap server atau sesi agent lain.

Sesi 4 — 20 Menit Praktik

Mengoprek Browser Tools

Buka file agents/research/tools/browser.ts

export const browsePage = functionTool({ name: "browse_web_page", description: "Buka halaman web dan baca kontennya", parameters: { type: "object", properties: { url: { type: "string", description: "URL yang mau dibuka" }, }, required: ["url"], }, handler: async ({ url }, ctx) => { const page = await ctx.sandbox.browser.navigate(url); const content = await ctx.sandbox.browser.getContent(); return `[${url}] — ${content.substring(0, 5000)}...`; }, });

Tool ini membuka URL menggunakan browser di dalam sandbox, mengambil konten halaman, dan mengembalikan teksnya kepada LLM untuk diproses.

Sesi 4 — Tugas 1 Praktik

Tugas: Tool Search & Browse

Buat tool yang bisa mencari di Google dan membaca artikel pertama — 10 menit

export const searchAndBrowse = functionTool({ name: "search_and_browse", description: "Search web & baca artikel pertama", parameters: { type: "object", properties: { query: { type: "string" }, }, required: ["query"], }, handler: async ({ query }, ctx) => { // Step 1: Search Google const searchUrl = `https://www.google.com/search?q=${encodeURIComponent(query)}`; await ctx.sandbox.browser.navigate(searchUrl); const results = await ctx.sandbox.browser.getContent(); // Step 2: Extract link pertama const linkMatch = results.match(/https?:\/\/[^\s"'<>]+/); if (!linkMatch) return "Tidak menemukan hasil."; // Step 3: Buka link & baca await ctx.sandbox.browser.navigate(linkMatch[0]); const content = await ctx.sandbox.browser.getContent(); return `[${linkMatch[0]}]\n\n${content.substring(0, 8000)}`; }, });

Sesi 5 — 20 Menit Praktik

File Processing

Membaca dan menghasilkan file dari dalam sandbox

Baca File (process_file)

export const processFile = functionTool({ name: "process_file", description: "Baca & proses CSV/JSON/TXT", parameters: { type: "object", properties: { filename: { type: "string" }, }, required: ["filename"], }, handler: async ({ filename }, ctx) => { const content = await ctx.sandbox.files.read(filename); if (filename.endsWith('.csv')) { const lines = content.split('\n'); const headers = lines[0].split(','); const data = lines.slice(1) .filter(l => l) .map(l => l.split(',')); return JSON.stringify({ headers, total: data.length, preview: data.slice(0, 3) }); } return content.substring(0, 5000); }, });

Generate File (generate_report)

export const generateReport = functionTool({ name: "generate_report", description: "Buat report TXT/CSV", handler: async ( { format, content }, ctx ) => { const filename = `report-${Date.now()}.${format}`; // Simpan ke sandbox await ctx.sandbox.files.write( filename, content ); // Simpan ke Blob Storage const store = getStore("reports"); await store.set(filename, content); return `Report siap: ${filename}`; }, });

Sesi 6 — 15 Menit Praktik

Shell Execution

Menjalankan command Linux di dalam sandbox yang terisolasi

Tool: run_shell

export const runShell = functionTool({ name: "run_shell", description: "Jalankan command Linux", parameters: { type: "object", properties: { command: { type: "string" }, }, required: ["command"], }, handler: async ({ command }, ctx) => { const result = await ctx.sandbox.commands.run(command); return `Output:\n${result.stdout || ''}\n${result.stderr || ''}`; }, });

const script = ` import json data = [ {"nama": "Andi", "nilai": 85}, {"nama": "Budi", "nilai": 92}, {"nama": "Cici", "nilai": 78} ] rata2 = sum(d["nilai"] for d in data) / len(data) print(json.dumps({ "rata_rata": rata2, "total": len(data), "data": data })) `; const result = await ctx.sandbox.runCode(script, "python3"); // Output: {"rata_rata": 85, "total": 3, "data": [...]}

Sesi 7 — 15 Menit

Session Memory

Membuat agent mengingat informasi lintas sesi percakapan

Tanpa Memory

Agent hanya ingat dalam 1 sesi. Jika halaman di-refresh, semua informasi hilang.

Dengan Memory (context.store)

Agent menyimpan dan mengambil informasi penting di setiap sesi baru. Auto-terhubung ke conversation_id.

EdgeOne menyediakan context.store — key-value storage yang otomatis terhubung ke conversation_id setiap user.

Sesi 7 — Lanjutan

Implementasi Session Memory

Simpan & Ambil Memori

// Simpan memori await ctx.store.set("user_name", "Rozak"); await ctx.store.set("preferences", JSON.stringify({ language: "id", tone: "santai", topic: ["AI", "Fullstack"] }) ); // Ambil memori const userName = await ctx.store.get("user_name"); const prefs = JSON.parse( await ctx.store.get("preferences") || "{}" );

Sesi 9 — 10 Menit

Production Readiness

Sandbox Timeout

Default: 5 menit idle lalu mati. Untuk proses berat, perpanjang dengan ctx.sandbox.setTimeout(600) (10 menit).

Error Handling

Sandbox bisa gagal (resource exhausted). Selalu wrap sandbox calls dalam try-catch. Berikan fallback yang informatif.

Rate Limiting

200K executions/bulan 100K GB-s/bulan sandbox

Observability

Buka /agent-metrics di mode development. Lihat trace: token usage, tool duration, memory consumption.

Keamanan

Jangan expose sandbox langsung ke user

Validasi input sebelum masuk sandbox

Set timeout yang wajar

Jangan hardcode credentials
